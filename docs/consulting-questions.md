# StoryQuant UX 컨설팅 질문 자료

> **목적**: 정보 밀도 높은 트레이딩 대시보드에서 개인 투자자에게 적합한 "서비스 최적점"을 찾기 위한 질문.

---

## 0. 한 줄 컨텍스트

뉴스를 룰 기반 **5개 분류** (호악재·실질성·카테고리·구체성·매체 신뢰도)로 자동 라벨링하고, 가격·매크로 데이터와 결합해 **종합 점수**로 KR/US 종목을 랭킹. **walk-forward 백테스트**로 신호 적중률을 측정해 사용자가 직접 검증할 수 있게 한다.

**현재 라이브**: https://likelihood-televisions-fcc-socks.trycloudflare.com/story_quant.html

---

## 1. 현재 명확히 결정한 것

| 항목 | 결정 |
|---|---|
| 타겟 사용자 | **개인 투자자** (1인 데이트레이더, 매크로 호기심형) — 기관 X, B2B X |
| 방향 | **기능 확장 중단 → 서비스 최적점** 추구 |
| 디자인 원칙 | 정보 과부하 회피, 핵심 신호 우선, "왜 움직였는지 1초" |
| ML 접근 | 룰 기반 + 통계 검증 (블랙박스 X) — 사용자가 IC IR 직접 확인 가능 |
| 검증 | walk-forward IR=+0.115 · 위기 시기 KR +3.05%/US +2.23% (win 70%+) |

## 2. 핵심 고민 4가지 — 컨설팅에서 답을 받고 싶은 부분

### 🔵 고민 1: 뉴스 "가공 수준"의 최적점

뉴스를 사용자에게 보여줄 때 어느 정도까지 가공해야 사용자가 신뢰하고 활용할 수 있는가?

| 수준 | 예시 | 사용자 인지 부담 | 신뢰감 | 의사결정 도움 |
|---|---|---|---|---|
| ① **Raw** | "삼성전자, 3분기 영업이익 컨센 상회" | 낮음 (그냥 헤드라인) | 높음 (원본 그대로) | ⚠ 낮음 (직접 판단해야) |
| ② **Labeled (현재)** | 위 + ◆호재·실질정보·실적·★★·연합1.00 | 중간 (라벨 5개 동시) | 중간 (기준 보임) | 중간 |
| ③ **Scored** | 상위 5개 종목 카드 + 종합 점수 +8.3 | 낮음 (압축됨) | 중간 (어떻게 계산?) | 높음 (바로 매수 후보) |
| ④ **Narrative** | "오늘 시장은 반도체 호재가 우세, 삼성 +3% 이끌어" | **가장 낮음** | ⚠ 낮음 (LLM 환각?) | 높음 (이해 쉬움) |

**질문**: 개인 투자자 대상이라면 ②와 ③ 중 어디가 더 적합한가? 혹은 ②→③→④로 점진 노출(Progressive Disclosure)?

---

### 🔵 고민 2: 백테스트는 어디에 두어야 하나

백테스트는 본인(개발자)의 **분석 타당성 증명** 도구로 만들었지만, 개인 투자자가 직접 walk-forward 모드를 토글하고 IC 행렬을 분석할까?

**옵션 비교**:

| 옵션 | 위치 | 사용자 부담 | 신뢰감 전달 | 본질 가치 |
|---|---|---|---|---|
| (a) 메인 메뉴 (현재) | 동등 위계 | **높음** (페이지 진입 부담) | 직접 검증 가능 | "전문가 도구" 느낌 |
| (b) "신뢰" 보조 메뉴 | 2단계 깊이 | 중간 | 진입 의향자만 봄 | 마케팅 도구 |
| (c) 작은 배지 노출 | 신호 옆 "검증된 적중률 +11.5%" 같은 1줄 | **낮음** | 즉시 인지 | 신뢰 배지 (요약 신호) |
| (d) Admin 전용 | 사용자 화면 X | 0 | 0 | 개발용 |

**질문**: 개인 투자자가 진짜 백테스트를 조작해서 의사결정에 사용할까? 혹은 (c) "이 신호 적중률 +0.115" 배지만으로 충분한가?

---

### 🔵 고민 3: KR + US 동시 노출 vs 단일 시장 집중

| 옵션 | 장점 | 단점 |
|---|---|---|
| (a) 1클릭 토글 (현재) | 두 시장 비교 쉬움 · 멀티 추적 사용자 만족 | 의사결정 부담 ↑ · 둘 다 가벼운 분석에 그칠 위험 |
| (b) 시작 시 시장 선택 | 한 시장에 집중 가능 | 매번 선택 부담 · 멀티 추적 사용자 이탈 |
| (c) 두 시장을 합쳐 한 화면 | 글로벌 매크로 흐름 한눈에 | 시간대·통화·거래시간 모두 다른데 직접 비교 의미? |
| (d) KR만 (US 제거) | scope 명확 | 미장 추적 사용자 포기 |

**질문**: 개인 투자자 중 KR과 US를 **동시**에 추적하는 비율이 얼마나 될까? 멀티시장이 차별화인가 분산인가?

### 🔵 고민 4: "처음 봐야 할 것"의 강제 hierarchy

데이터가 많은 대시보드에서 사용자가 첫 5초에 무엇을 보는가에 따라 서비스 가치 결정.

**현재 (Newsroom-first)**: 인덱스 스트립 + 매크로 + 종합 신호 sweep + 헤드라인 stream 동시 표시 → 사용자가 어디부터 봐야 할지 부담.

**대안**:

- (a) 매크로 분위기 한 줄만 ("오늘 시장: 위기 시기, 방어주 우세")
- (b) Top-3 종목 카드 만 ("오늘 가장 강한 신호: 삼성/SK하이닉스/NVDA")
- (c) "지금 가장 중요한 1가지" 큰 카드 1개
- (d) 자유 탐색 (지금) — 사용자가 직접 선택

**질문**: 모바일 우선 디자인에서 첫 화면에 정보 hierarchy를 어떻게 강제해야 사용자가 이탈하지 않고 깊이 진입하는가?

## 5. 추가 고민 — 신뢰도(Trust)는 어떻게 보여줘야 하나

분석 모델·결론의 신뢰도를 사용자에게 어떻게 전달할지가 가장 큰 고민. 너무 명시적이면 "AI 점쟁이" 인상, 너무 무딘 표현은 가치 X.

### 5.1 신뢰도 표시 6가지 패턴 (mock 별첨)

| # | 패턴 | 예시 | 장점 | 단점 |
|---|---|---|---|---|
| A | 절대 점수 | "신뢰도 87%" · "IR=+0.115" | 정확 · 비교 가능 | "어떻게 계산?" · 학술 거부감 |
| B | 별 등급 | ⭐⭐⭐ (3성) | 친숙 · 모바일 친화 | 깊이 없음 · "왜 3성?" 모호 |
| C | 자연어 라벨 | "강한 신호" · "참고용" | 즉시 이해 | 기준 모호 |
| D | 적중률 막대 | ▓▓▓▓▓▓▓░░ 70% | 시각적 | 시점·조건 의존 |
| E | 구체 사례 | "10번 중 7번 적중 (위기 시기)" | 솔직 · 구체적 | 조건 명시 필요 |
| F | 반례 노출 | "✓ 7건 / ✗ 2건 (실패 사례 보기 →)" | 신뢰 강화 (역설적) | 위험감지 사용자 부담 |

### 5.2 추천 방향: Multi-level Trust (점진 공개)

```
[Level 1 즉시]      신호 카드 작은 배지   →   "✓ 검증된 신호 · 70%"
                    바쁜 사용자 · 1초 인지     (1줄)
                    ↓ 클릭
[Level 2 호기심]    작은 모달               →   "위기 시기 +3.05%, 평온기 0%,
                    "왜 70%?" 알고 싶은 사람    최근 90일 적중 28/40"
                    ↓ "직접 검증" 버튼
[Level 3 전문가]    백테스트 페이지         →   IC · IR · regime · decile
                    신호 자체 검증              자기 모드 토글
```

**이유**: 사용자 수준에 따라 정보량 차등 노출. Level 1만으로 모바일 충족, Level 3로 전문가도 잡음.

### 5.3 핵심 결정 필요한 3가지

1. **위기/평온 구분 명시할까?** — 솔직하면 "평온기엔 alpha 없음" → 사용자 이탈 위험 vs 신뢰↑
2. **반례를 보여줄까?** — 못 맞춘 사례 노출이 신뢰 강화 vs 자기파괴
3. **숫자 vs 라벨?** — IR=0.115는 모르고 ⭐⭐⭐도 모호. **"70% 적중률 (위기 시기)"** 같이 자연어 + 숫자 + 조건 명시가 정답?

### 5.4 컨설팅에서 받고 싶은 답변

- 6가지 패턴 중 개인 투자자에게 어떤 게 가장 적합?
- Multi-level 점진 공개 방향이 맞나? 아니면 한 가지 패턴으로 통일?
- "위기 시기에만 alpha"라는 솔직한 디스클로저가 신뢰 자산인가, 이탈 트리거인가?
- 반례(실패 사례) 노출은 fintech 일반 사용자에게 부담? 도움?
