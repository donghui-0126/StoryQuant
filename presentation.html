Product Goal

KR · US 시장의 단일 종목 또는 시장 전체 가격 움직임 뒤에 있는 뉴스 흐름을 자동으로 분류·집계·랭킹하여,
개인 투자자가 1초 안에 "왜 움직였는지"를 파악하고
30~120일 과거 데이터로 검증된 신호로 의사결정할 수 있게 한다.

사용자가 얻는 가치

  • 시간 절감: 수천 건 헤드라인 스크롤 → 1 화면 흐름 요약
  • 의사결정 지원: 종합 점수 상위 = 매수 후보
  • 신뢰: 룰 기반 + 적중률 검증 (블랙박스 X)
  • 시장 확장: KR ↔ US 한 클릭 (어댑터 분리)
  • 검증 가능성: 과거 백테스트로 초과 수익 사전 측정

핵심 사용 시나리오 (3가지)

  1. 아침 브리핑 (장 시작 전)
    → 헤드라인 모니터에서 32개 매체 신선 뉴스 + 호악재/카테고리 자동 분류 확인. 시장 분위기 (위기 / 평온) 진단.
  2. "왜 움직였지?" (장중)
    → 종목 클릭 → 종목별 뉴스 ("이미 반영" 플래그 포함) + 차트 + 종합 신호. 섹터 뉴스 → 테마 회전 모니터로 관련 종목 확장.
  3. 신호 검증 (주말)
    → 과거 백테스트 (8 모드) → 적중률·위험 대비 수익·분위기별 초과 수익·상위/하위 그룹 수익 차 확인 후 다음 주 전략 결정.

3가지 핵심 "연결" — 막연한 매핑이 아닌 실제 구현 결정

고객 니즈 → ML framing

"정보 과부하"가 진짜 problem이면 헤드라인을 단순 list가 아닌 자동 라벨링 stream으로 설계. → enrich_article() 한 함수에서 5개 분류 task 동시 적용.

데이터 한계 → 접근법 선택

라벨링된 학습 데이터 없음 → LLM/FinBERT 대신 약한 라벨 룰 기반으로 빠른 초기 버전. → 백테스트로 신호 적중률·초과 수익 직접 측정 → 데이터 라벨링 투자 의사결정 근거 확보.

평가 지표 → 화면 구조 결정

사업 지표 = "사용자가 신호 신뢰" → 모델 지표 노출이 화면의 핵심. → 백테스트 페이지가 곧 모델 지표 대시보드. 사용자가 직접 적중률 확인.

→ 9단계 매핑 전체 표는 B 백업 A4.

현재 가장 큰 3개 애로사항

전문가 trouble-shooting을 받고 싶은 우선순위 3개. 전체 6개 표는 백업 A4.

모델 #1

신호 적중률 정체

현재 호악재 신호 적중률 +0.115 (0=무작위, 0.3+=강함). 목표 0.3 이상.

가설: 룰 기반 실질성 분류가 "매수 상향" 같은 애널리스트 의견을 실질 정보로 오분류.

시도: 키워드 사전 확장 + 시간대 가중 — 효과 미미.

필요한 도움 — FinBERT 의사 라벨로 실질성 분류 정밀도 검증 방법. active learning(능동 학습) 활용 가능?

Data #2

Historical 뉴스 콜드런

4년치 종목별 fetch 첫 콜 ~30분.

Naver scraping · Google News rate limit (TOS 회색). 백테스트 첫 클릭이 timeout.

시도: 24h memory + 7d disk 캐시. ThreadPool 14 worker.

필요한 도움 — 합법적 historical 라이선스 옵션 (DART · Refinitiv 대안), 캐시 워밍 자동화.

UX #3

3.7MB SPA 분해

초기 로드 3~5초 (모바일 4G).

public/{css,js}로 분리 시작했으나 main HTML 아직 monolith.

시도: lazy-render CSS · INSTANT-HIDE · PWA cache.

필요한 도움 — import-map · streaming HTML 전략. 모바일 4G LCP < 2.5s 가능?

→ 검증·성능·기획 등 나머지 3개 항목은 B 백업 A4.

Q&A 백업 A2 · "이미 반영된 거 아냐?" 반박

"뉴스는 이미 가격에 반영" — 시장은 정말 효율적인가?

질문은 정당. 효율적 시장 가설(공개 정보는 즉시 가격에 반영된다는 이론) 이 강하게 성립하면 우리 신호 적중률은 0이어야 한다. 그러나 학술 · 실측 증거는 일관되게 부분적 비효율을 가리킨다.

학술적 근거 — 효율적 시장 가설 반례 4가지

연구발견
Bernard-Thomas 1989
실적 후 가격 드리프트		 실적 발표 후 60~90일간 가격이 계속 같은 방향으로 흐름. 어닝 서프라이즈 클수록 크다. 35년 후 재현됨
Tetlock 2007
뉴스 톤 예측력		 WSJ 칼럼의 부정적 어조 → 다음날 다우지수 하락 예측력
Da-Engelberg-Gao 2011
검색량 주의력		 Google 검색량 = 개인 투자자 주의력 지표. 검색 ↑ → 2주 내 상승, 1년 후 되돌림
Hirshleifer-Teoh 2003
한정된 주의력		 투자자의 한정된 주의력 → 정보 즉시 반영 안 됨. 정보 처리 비용 자체가 초과 수익의 원천

우리 코드의 4가지 답

① "이미 반영" 자동 감지

serve/core/classify.py:199-244. 뉴스 직전 5일 가격 변동 측정 → 호재 뉴스 + 5일간 ≥+5%면 priced_in=True → 종합 점수에서 페널티 -2.0×비율. "이미 반영된" 신호를 명시적으로 식별·제외하는 코드.

② 실질 정보 vs 결과 보도 분리

"급등·강세·신고가" (결과 보도) = 가격 결과 묘사, 이미 반영. "실적·M&A·FDA·수주" (실질 정보) = 신선 정보. 백테스트에서 실질 정보 전용 모드 별도 제공.

③ 백테스트 적중률 +0.115 (호악재 신호)

완전 효율적이면 적중률 0. 실측 +0.115 (KR), +0.094 (US) = 약하지만 측정 가능한 초과 수익. 미래 참조 없는 과거 시점별 검증.

④ 시장 분위기별 초과 수익

평온기 (VIX≤15) 초과 수익 ≈ 0 — 효율적 시장 가설 성립. 위기 시기 (VIX≥22) 초과 수익 +3.05% KR / +2.23% US (10번 중 7번+ 적중) — 깨짐. 우리는 깨지는 시기·종목을 식별.

효율적 시장 가설은 평균적으로는 성립하지만, 정보 처리 능력의 한계(한정된 주의력)와 행동편향(과소 반응)이 누적되는 특정 시기·특정 종목에서 깨진다. StoryQuant의 가치는 "모든 뉴스에서 초과 수익을 찾자"가 아니라 "초과 수익이 존재하는 좁은 영역(실질 정보 × 위기 시기 × '이미 반영'되지 않은)을 식별하고 그곳에 집중하자"는 것이다. 우리 코드는 그 식별을 자동화한다.

  1. 장 시작 전 — 빨간 막대 다수 헤드라인 = 위험. 파란 ◆ 클러스터 = 실질 호재

  3. 카테고리 필터 "거시·지정학" → 매크로 흐름만 추출 → 시장 분위기 진단
  4. ★0 + ○ 결과보도 클러스터는 무시 → 노이즈 cut

  2. 섹터 뉴스 → 종목 매핑 — 헤드라인에서 "2차전지株 약세" 보이면 → 2차전지 테마 카드 → 어느 종목이 가장 빠졌는지 확인

현재 한계 (정직)

30종목 외 라이브 가격 없음. 테마 바스켓 중 30종목과 안 겹치는 종목은 mock drift — 발표 시 명시.

Q&A 백업 A7 · 백테스트 — 사용자가 직접 신호 검증하는 곳

§백테스트 — 8 모드 신호 검증 도구

📸 캡처 · #walkforward ↗ 라이브 새창

무엇

사용자가 8개 신호 모드 중 하나 선택 → 과거 N일 시점별로 미래 참조 없이 시뮬레이션 → 신호 적중률 · 위험 대비 수익 · 시장 분위기별 초과 수익을 한 페이지에서 확인.

이 페이지가 곧 "모델 지표 대시보드" — 사업 지표(신뢰)를 위해 모델 지표 노출이 화면의 핵심이라는 IA 결정의 결과물.

8 모드 — 각각 다른 신호 조합

  • 가격만 — 5일/20일 모멘텀만 (뉴스 X). 베이스라인
  • 뉴스만 — 호악재 균형 + 뉴스 빈도만
  • 혼합 (디폴트) — 가격·뉴스·거래량 가중 합산
  • 실질 정보 — substantive 헤드라인만 사용
  • 주의력 — 최근 3일 집중된 뉴스 비율
  • 정보 — 신뢰 + 구체성 + 호악재 곱
  • 분위기별 호악재 — VIX ≥22면 호악재 가중↑, ≤15면 cash

결과 패널 5가지

  1. 누적 수익 곡선 vs 벤치마크 (KOSPI/S&P) — 가장 큰 정보
  2. 신호 적중률 행렬 — 14개 신호 각각의 적중률·hit rate. "어느 신호가 진짜 작동하나?"
  3. 신호 간 중복도 — 어느 신호가 서로 같은 정보 담고 있나 (직교성 확인)
  4. 분위기별 분해 — 위기 / 평온 / 중립 각각의 초과 수익
  5. 카테고리별 반감기 — 실적·M&A·임상 등 카테고리별 forward return + std

어떻게 사용 (Decision flow)

  1. 모드 "혼합" 선택 → "Apply" → 5분 대기 (캐시 hit면 즉시)
  2. 누적 수익 곡선이 벤치마크 위면 통과. 아래면 모드 변경
  3. 분위기별 분해 확인 → 평온기 초과 수익 ≈ 0이면 risk_off 시기에만 운용
  4. 적중률 행렬에서 양수 신호 골라서 다음 모드 만들 때 weight 조정
Q&A 백업 A8 · 종목 상세 — 차트 + 종목별 뉴스 + "이미 반영" 플래그

종목 상세 — 헤드라인 클릭 → 같은 화면 sticky panel

📸 캡처 · 종목 클릭 후 sticky panel (예: 006260) ↗ 라이브 (헤드라인 클릭)

무엇 · 진입 경로

헤드라인 모니터의 종목 태그나 테마 회전 카드의 대표 종목 칩을 클릭하면 같은 화면에 sticky panel로 종목 상세가 열림. ESC 또는 닫기 버튼으로 닫힘.

구성 요소 (3개 패널)

  1. 차트 — Yahoo Finance v8 chart API. 모드 토글: LINE / CANDLE / VOL + MA20/60/RSI 오버레이. range 1d ~ 10y. 실제
  2. 종목별 뉴스 — 종목 전용 API. m.stock.naver.com/api/news/stock/{code} (KR) / news.google.com/rss/search?q={ticker}+stock (US). 한 페이지당 20건, 5분류 자동 라벨. 실제
  3. 시세 + 메타 — 실시간 가격, 시총, PER, 외인 지분율 (KR=Naver scraping). 실제 KR · Stub US 시총

핵심 차별점 — "이미 반영" 플래그 ⚡

serve/core/classify.py:199-244 · 종목 뉴스 각각에 priced_in 자동 판별:

  • 호재 뉴스 + 직전 5일 ≥+5% 오른 종목 → ⚡ priced_in
  • 악재 뉴스 + 직전 5일 ≤-5% 내린 종목 → ⚡ priced_in
  • 종합 점수에서 페널티 적용 (-2.0 × ratio)

→ "이미 다 오른 거 뒷북 매수" 방지 기능.

어떻게 사용 (Decision flow)

  1. 헤드라인에서 ◆ 실질 정보 + 종목 태그 발견 → 클릭
  2. 차트로 직전 5일 추세 확인 (이미 +5% 이상이면 ⚡ 플래그 떠 있을 것)
  3. 종목 뉴스 리스트에서 ⚡ 비율 확인 — 50% 이상이면 "이미 다 반영"
  4. 실질 정보(◆) 중 ⚡ 없는 뉴스만 새 catalyst — 매수 후보

⚠ 현재 한계 (정직)

  • 데이터 소스 비대칭 — KR=Naver mobile API (한국 매체 통합) / US=Google News 검색. 같은 종목이라도 양 시장 결과 quality·분류가 다름
  • 본문 부재 — RSS title + summary만. 본문 분석 불가 → 분류 정밀도 제한 (애로사항 #1 모델 IR 정체의 일부 원인)
  • 섹터 뉴스 누락 — "2차전지株 약세" 같이 종목명 없는 뉴스는 종목별 API에 안 잡힘 (테마 회전에서만 잡힘)
  • 동음이의 노이즈 (US) — Google 검색이라 "Apple" → 회사 vs 과일, "Tesla" → 회사 vs 발명가 노이즈
  • Historical 깊이 — Naver mobile API는 최근 100건만. 4년치 fetch는 Google News date-range로 별도 (느림 · 첫 콜 30분 — 애로사항 #2)
  • 법적 회색지대 — Naver scraping은 robots.txt 모호 · Google News는 TOS 회색. v2에서 정식 라이선스 검토 필요
  • US 시총 stub — sticky panel의 시총·PER·외인% 메타가 KR만 실측, US는 빈 값
